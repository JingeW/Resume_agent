# Template Placeholder Specification

This file documents every `<<PLACEHOLDER>>` in `jakes_resume.tex`
and the exact string format the renderer must produce for each.

---

## Header

| Placeholder       | Format                              | Source                        |
|-------------------|-------------------------------------|-------------------------------|
| `<<NAME>>`        | Plain string, e.g. `Alex Chen`      | identity.json → name          |
| `<<PHONE>>`       | Plain string, e.g. `(+1) 555-123-4567` | identity.json → phone      |
| `<<EMAIL>>`       | Plain string, e.g. `alex@example.com` | identity.json → email      |
| `<<LINKEDIN_URL>>`| Full URL                            | identity.json → linkedin_url  |
| `<<GITHUB_URL>>`  | Full URL                            | identity.json → github_url    |
| `<<SCHOLAR_URL>>` | Full URL                            | identity.json → google_scholar_url |

| Placeholder   | Format                            | Source                                    |

| `<<SUMMARY>>` | Plain paragraph, no LaTeX markup  | Generated by agent from identity.json → summary_variants |

## Core Competencies

| Placeholder        | Format | Example |
|--------------------|--------|---------|
| `<<SKILLS_LINES>>` | One `\textbf{Category}{: item1, item2, item3} \\` per line | See below |

Example output:
```latex
\textbf{Engineering}{: Python, SQL, AWS, Docker, Git, pandas, scikit-learn, PyTorch} \\
\textbf{AI \& ML}{: LLMs, prompt engineering, RAG, multimodal modeling, model evaluation} \\
\textbf{Analytics}{: experiment design, A/B testing, statistical validation, error analysis} \\
\textbf{Collaboration}{: cross-functional delivery, stakeholder communication, mentoring}
```

Note: Last line has no trailing `\\`.
Note: Ampersand in category name must be escaped: `AI \& ML`.

## Experience

Each position uses three placeholders for metadata and one for bullets.

| Placeholder       | Format                          |
|-------------------|---------------------------------|
| `<<EXP1_COMPANY>>`| Company name, e.g. `Polygon Health Analytics LLC` |
| `<<EXP1_LOCATION>>`| Location string, e.g. `Remote` |
| `<<EXP1_TITLE>>`  | Job title, e.g. `Data Scientist` |
| `<<EXP1_DATES>>`  | Date range, e.g. `Oct 2024 -- Sep 2025` |
| `<<EXP1_BULLETS>>`| One `\resumeItem{...}` per line |

EXP1 = PHA (most recent), EXP2 = Postdoc, EXP3 = PhD/GRA.

Example bullets block:
```latex
\resumeItem{Built an OCR+LLM pipeline (AWS Textract + GPT-4o) extracting structured pathology indicators from 60,000+ clinical PDFs, achieving 95\% accuracy.}
\resumeItem{Designed schema validation and retry logic for production reliability.}
```

Note: Percent signs must be escaped as `\%`.
Note: Do not add extra blank lines between \resumeItem entries.

## Projects

| Placeholder        | Format |
|--------------------|--------|
| `<<PROJECTS_BLOCK>>`| One project block per selected project (see below) |

Each project block:
```latex
\resumeProjectHeading
    {\textbf{MAGIC} $|$ \emph{Multimodal Analysis of Genomics, Imaging and Clinical Data}}
    \resumeItemListStart
      \resumeItem{Built an OCR+LLM pipeline...}
      \resumeItem{Integrated pathology-derived features...}
      \resumeItem{Designed schema validation...}
    \resumeItemListEnd
```

Note: Title format is `\textbf{Short Name} $|$ \emph{Full Subtitle}`.
Note: No date on the right side (unlike original Jake's template) — we removed it to save space.

## Education

| Placeholder          | Format |
|----------------------|--------|
| `<<EDUCATION_BLOCK>>`| One `\resumeSubheading` block per degree |

Example:
```latex
\resumeSubheading
  {West Virginia University}{Morgantown, WV}
  {Ph.D. in Computer Science}{Aug 2016 -- May 2023}
\vspace{-3pt}\item[] \small\textit{Specialization in Artificial Intelligence, Data Analysis, and Signal Processing.}
    \resumeSubheading
  {West Virginia University}{Morgantown, WV}
  {M.S. in Statistics}{Aug 2013 -- Dec 2015}
\vspace{-3pt}\item[] \small\textit{Focus on Statistical Modeling, Regression, and Experimental Design.}
    \resumeSubheading
  {Anhui University of Finance and Economics}{Anhui, China}
  {B.M. in Accounting}{Sep 2007 -- Jul 2011}
\vspace{-3pt}\item[] \small\textit{Completed foundational coursework in management and accounting.}
```

Note: Education entries do not have bullet points — no `\resumeItemListStart` block.
Note: A `note` field in `education.json` renders as `\vspace{-3pt}\item[] \small\textit{note}` — close to its degree entry with natural spacing below.

## Publications

| Placeholder             | Format |
|-------------------------|--------|
| `<<PUBLICATIONS_LIST>>` | One `\item \small{...}` per publication |

Example:
```latex
\item \small{\textbf{A Chen}; et al. ``Example Paper Title on Clinical AI.'' \textit{Nat. Methods}. (2025)}
\item \small{B Smith; \textbf{A Chen}; et al. ``Another Example Paper Title.'' \textit{Cell Reports}. (2024)}
```

Note: Use curly double quotes: `` `` for open, '' for close.
Note: Venue in `\textit{}`.
Note: The candidate's abbreviated name (first initial + last name, e.g. `A Chen`) is wrapped in `\textbf{}` wherever it appears in the author string, e.g. `\textbf{A Chen}; et al.`

## Special Characters to Escape

| Character | LaTeX escape |
|-----------|-------------|
| `%`       | `\%`        |
| `&`       | `\&`        |
| `#`       | `\#`        |
| `_`       | `\_`        |
| `~`       | `\textasciitilde` |
| `^`       | `\textasciicircum` |
| `+`       | `+` (no escape needed in text mode) |

The renderer must escape all of the above in any string sourced from profile JSON
before inserting into the template.
